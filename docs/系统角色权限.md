# 系统角色权限说明

## 1. 角色设计原则

- 角色来源于学校实际岗位，不凭空设计
- 菜单权限、按钮权限、数据权限分层控制
- 同一用户允许拥有多个角色
- 角色权限以最小必要原则配置
- 管理员拥有配置能力，但业务数据查看应保留审计记录

## 2. 核心角色说明

### 学生

- 功能权限：提交申请、上传材料、查看本人流程、接收通知
- 数据权限：仅可查看本人数据

### 班主任

- 功能权限：审批班级学生事务、维护异常学生材料、发布班级通知
- 数据权限：仅可查看所带班级学生相关数据

### 辅导员

- 功能权限：审批学生事务、查看学生预警、管理班级事务
- 数据权限：按所负责班级或专业查看

### 任课教师

- 功能权限：发起调课、提交教学材料、申报科研项目、查看本人流程
- 数据权限：仅可查看本人发起的数据和本人课程相关数据

### 教研室主任/秘书

- 功能权限：审核课程标准、教材征订、教学材料
- 数据权限：按教研室范围查看教学类数据

### 学院教学秘书

- 功能权限：汇总学院教学流程、查看报表、配合业务管理
- 数据权限：按本学院查看教学数据

### 学院领导

- 功能权限：审批学院层面的重要流程、查看学院统计报表
- 数据权限：按本学院查看相关业务

### 教务处

- 功能权限：审批和管理教学教务类流程、维护教学类规则、查看全校教学报表
- 数据权限：可查看教学教务相关全校数据

### 科技处

- 功能权限：管理课题申报、组织评审、维护科研项目状态
- 数据权限：可查看科研管理相关全校数据

### 学工处

- 功能权限：管理奖助学金、困难认定、学生事务统计
- 数据权限：可查看学生事务相关全校数据

### 后勤/宿管/资产管理员

- 功能权限：处理报修、宿舍、会议室、物资、资产等业务
- 数据权限：可查看所属业务域相关数据

### 评审专家

- 功能权限：参与指定项目或材料评审
- 数据权限：仅查看分配给本人的评审数据

### 系统管理员

- 功能权限：维护用户、角色、组织、菜单、字典、参数、流程模板、日志
- 数据权限：以系统配置管理为主，业务数据操作需审计

## 3. 权限模型

### 菜单权限

- 控制用户是否能进入某个模块页面
- 例如：学生不可进入系统管理，教师不可进入管理员配置页

### 按钮权限

- 控制用户是否可以执行新增、编辑、删除、审批、导出等操作
- 例如：班主任可审批学生请假，但学生不可审批

### 数据权限

- 控制用户能看到哪些组织范围内的数据
- 维度包括学校、学院、专业、班级、个人

## 5. 数据权限控制建议

- 学生：仅本人
- 班主任/辅导员：按带班范围
- 任课教师：按本人课程和本人发起
- 教研室：按教研室归属
- 学院：按学院归属
- 教务处/科技处/学工处：按业务域全校范围
- 后勤/宿管/资产：按业务域和资源归属范围
- 管理员：以配置维护为主，不默认开放所有业务明细

## 6. 权限实施要求

- 所有业务接口必须校验当前用户角色和数据范围
- 页面菜单与后端接口权限必须同时控制
- 审批类操作必须保留审批人、时间、意见和节点信息
- 导出、批量操作、配置修改等敏感行为必须记录日志

## 7. 当前一期已落地权限边界

当前代码已将权限控制落到数据库和后端接口，具体边界如下：

- 系统管理员：
- 可访问用户管理、组织管理、角色权限管理、附件中心、流程模板管理
- 可维护角色菜单和按钮权限映射
- 普通业务角色：
- 继续按业务域菜单进入学生事务、教学教务、科研管理、行政后勤
- 不可直接访问系统管理相关接口和页面
- 菜单权限：
- 登录后根据 `sys_role_menu + sys_menu` 返回顶级菜单
- 系统管理子页面按 `system` 菜单入口统一收口
- 按钮权限：
- 通过 `sys_permission + sys_role_permission` 维护按钮/接口权限码
- 当前一期先由管理员角色完整持有系统管理按钮权限
- 数据权限：
- 业务详情继续按“申请人 / 当前审批人 / 已处理人 / 管理员”判断
- 附件查看按申请单可读权限控制
- 附件上传按“申请人 / 当前审批人 / 管理员”控制
- 附件删除按“上传人 / 管理员”控制

上述规则用于支撑当前一期最小可用底座。学院、班级、教研室等更细粒度数据范围控制，后续仍需在组织数据域进一步细化。
## 8. 业务归属修订说明

- 学生生命周期业务统一包括请假、销假、离校、返校、在读证明、奖助学金、困难认定以及毕业设计开题和中期检查。
- 毕业设计开题申请和毕业设计中期检查由学生发起，指导老师或相关教师审批，不归为教务处主导的教学教务流程。
- 科研成果登记继续归入科研管理，由科研相关角色处理。

## 9. 第 7 批权限边界补充

- 车辆申请：教师可发起并查看本人申请，管理员负责审批和排车。
- 班级通知回执：班主任可发起和确认发布，学生可查看相关通知入口，管理员可审计查看。
- 学生预警处理：班主任可登记和处理所负责学生预警，学工处复核处理结果，管理员可查看。
- 证明材料补交：学生可发起本人材料补交，班主任复核补交材料，管理员可查看。
- 公告发布：教师可提交公告发布申请，管理员审核发布；其他业务角色按菜单进入后查看有权限范围内记录。

接口权限继续遵循后端校验优先原则：

- 未登录用户不能访问第 7 批业务接口。
- 申请人、当前审批人、已处理人和管理员具备详情可读权限。
- 审批动作必须由当前审批人执行，不能通过修改申请 ID 越权审批。
- 前端菜单隐藏只作为体验控制，不替代后端权限校验。
