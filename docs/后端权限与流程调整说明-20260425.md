# 后端权限与流程调整说明

## 1. 说明范围

本说明用于补充 2026-04-25 对以下 5 个业务的后端调整：

- 学生离校申请
- 学生返校确认
- 毕业设计开题申请
- 毕业设计中期检查
- 科研成果登记

本次调整不引入新的工作流引擎，不更换技术栈，继续复用现有通用申请与审批能力。

## 2. 权限与审批调整

### 2.1 学生离校申请

- 仍由学生发起。
- 提交前增加时间校验，结束时间必须晚于开始时间。
- 后端新增“未闭环且时间重叠离校申请”校验，避免重复离校记录并行存在。
- 当前审批角色仍为 `ADVISER`。

### 2.2 学生返校确认

- 仍由学生发起。
- 返校确认必须关联本人已审批通过的离校申请单号。
- 后端会校验该离校单是否属于当前学生，且是否已审批通过。
- 同一离校单在 `DRAFT`、`PENDING`、`IN_PROGRESS`、`APPROVED` 状态下不允许重复创建返校确认。
- 返校审批优先复用关联离校申请的实际审批人，避免返校单随机落到其他班主任。

### 2.3 毕业设计开题申请

- 归属学生生命周期业务，由学生发起。
- 提交时必须填写指导老师姓名。
- 后端会校验指导老师是否存在、是否启用、是否具有 `TEACHER` 角色。
- 审批人不再按 `TEACHER` 角色随机选取，而是按开题记录中的指导老师精确匹配。
- 若指导老师重名，系统拒绝自动提交，避免错误分派审批人。

### 2.4 毕业设计中期检查

- 归属学生生命周期业务，由学生发起。
- 提交时要求进度比例在 `0` 到 `100` 之间。
- 中期检查前必须已存在同一学生、同一课题、且已审批通过的开题申请。
- 中期检查日期不得早于开题日期。
- 审批人按已通过开题记录中的指导老师匹配，不再按教师角色随机分派。

### 2.5 科研成果登记

- 仍归科研管理业务，由教师发起、科技处处理。
- 后端补充登记日期格式校验，确保字段和数据库类型一致。

## 3. 流程状态同步调整

为避免 `wf_application.status` 与业务表 `business_status` / `flow_status` 脱节，后端已将以下业务表纳入统一状态同步：

- 教案提交
- 教学大纲提交
- 成绩更正申请
- 考试安排申请
- 教室借用申请
- 学生离校申请
- 学生返校确认
- 毕业设计开题申请
- 毕业设计中期检查
- 科研成果登记

同时补齐了此前遗漏的其他扩展业务表状态同步。

## 4. 初始化脚本一致性调整

为保证新环境初始化可执行，数据库脚本已补齐以下字段：

### 4.1 `wf_node_definition`

- `approver_role_code`
- `status`
- `updated_at`

### 4.2 `file_attachment`

- `content_type`
- `deleted_flag`
- `updated_at`

以上字段已被现有后端代码使用，因此属于脚本与实现的对齐修复。

## 5. 当前仍保留的边界

- 学生与班主任、学生与指导老师的正式组织关系表当前仍未独立建模。
- 本次先通过“关联离校审批人复用”和“指导老师姓名精确匹配”降低错误分派风险。
- 后续若继续细化数据权限，建议新增学生与班主任、学生与指导老师的关系映射表，彻底收敛审批人与可见范围。
