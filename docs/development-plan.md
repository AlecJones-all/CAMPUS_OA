# 校园 OA 开发计划

## 1. 开发阶段

### 第一阶段：需求与方案定稿

- 输出需求分析说明书
- 输出角色权限矩阵
- 输出核心业务流程清单
- 输出数据库设计方向
- 输出前后端模块划分

完成验证：
- 业务域覆盖学生、教学、科研、后勤四大主线
- 一期范围和二期预留边界明确
- 关键角色和关键流程无缺失

### 第二阶段：工程基线

- 创建后端 Spring Boot 工程
- 创建前端 Vue 3 工程
- 建立公共返回结构、基础路由、工作台框架
- 创建数据库初始化脚本

完成验证：
- 后端可以启动健康检查接口
- 前端可以启动并展示基础导航页面
- 数据库脚本可成功创建核心基础表

### 第三阶段：平台底座

- 登录与认证
- 用户、角色、组织、岗位管理
- 菜单权限与数据权限
- 字典和系统参数
- 附件中心
- 消息中心

完成验证：
- 管理员可以完成用户、角色和组织维护
- 权限可以控制页面和接口访问
- 附件可上传并与业务单据关联

### 第四阶段：流程中心

- 流程模板
- 流程实例
- 待办、已办、抄送
- 审批、退回、撤回、转交
- 流程轨迹

完成验证：
- 通用流程引擎可以支撑多个业务类型
- 全链路流程状态清晰可追溯

### 第五阶段：一期业务模块

- 请假销假
- 实习材料
- 异常学生材料
- 教师调课
- 课程标准评审
- 课题申报
- 会议室申请
- 维修工单

完成验证：
- 每个模块具备录入、审批、查询、统计基础能力
- 不同角色看到的数据范围正确

### 第六阶段：报表与优化

- 超时预警
- 提交率统计
- 审批时效统计
- 角色工作台优化
- 二期扩展点整理

完成验证：
- 报表数据可按学院、时间、状态筛选
- 超时任务和异常流程可被识别

## 2. 技术栈基线

- 前端：`Vue 3 + Vite + TypeScript + Element Plus`
- 前端配套：`Pinia + Vue Router`
- 后端：`Spring Boot`
- 后端配套：`Spring Security + MyBatis-Plus`
- 数据库：`MySQL`
- 文档：`Markdown`

## 3. 文档结构建议

- `requirements.md`：总体需求与业务边界
- `development-plan.md`：阶段计划与交付标准
- 后续补充：
  - `role-permission-matrix.md`
  - `business-flows.md`
  - `api-design.md`
  - `prototype-list.md`

## 4. 前后端模块划分

### 前端

- `layouts`：整体框架布局
- `router`：路由和权限入口
- `stores`：用户、菜单、待办等状态管理
- `views/dashboard`：工作台
- `views/student-affairs`：学生事务
- `views/academic`：教学教务
- `views/research`：科研管理
- `views/logistics`：行政后勤
- `views/system`：系统管理

### 后端

- `common`：公共模型、返回结构、异常
- `security`：认证鉴权
- `system`：用户、角色、组织、字典、参数
- `workflow`：流程定义、流程实例、审批记录
- `file`：附件中心
- `message`：消息与公告
- `studentaffairs`：学生事务
- `academic`：教学教务
- `research`：科研管理
- `logistics`：行政后勤

## 5. 数据表设计方向

- 平台基础表：用户、角色、组织、岗位、菜单、字典、参数、日志
- 流程引擎表：流程定义、节点定义、流程实例、任务实例、审批记录、抄送记录
- 业务通用表：业务主单、业务附件、业务意见、业务操作日志
- 业务域表：请假单、实习材料、异常学生记录、调课申请、课程标准评审、课题申报、会议室申请、维修工单

## 6. 每一步验证方式

- 文档：进行需求评审，确认角色、流程、字段、边界
- 后端：启动成功，健康检查接口返回正常
- 前端：页面可访问，路由切换正常
- 数据库：建表脚本执行成功，主外键与索引合理
- 模块：按业务场景跑通提交流程和状态流转
- 权限：不同角色登录验证页面和数据隔离
- 报表：基础统计结果与业务台账一致
