# 校园 OA 开发计划

## 1. 开发目标

本计划面向校园 OA 一期建设，目标是在本地完成需求文档、技术基线、数据库设计、系统底座和高频业务模块开发，并逐步推进到可演示、可验证、可继续扩展的状态。

## 2. 开发原则

- 先文档后实现
- 先底座后业务
- 先高频后低频
- 先共性能力后个性流程
- 每一步必须有明确完成标准

## 3. 技术栈基线

- 前端：`Vue 3 + Vite + TypeScript + Element Plus`
- 前端配套：`Pinia + Vue Router`
- 后端：`Spring Boot`
- 后端配套：`Spring Security + MyBatis-Plus`
- 数据库：`MySQL`
- 文档：`Markdown`

## 4. 阶段划分

### 第一阶段：需求与方案阶段

工作内容：

- 完成需求分析文档
- 完成需求调研计划
- 完成业务流程清单
- 完成系统角色权限说明
- 完成数据库设计草案
- 明确技术栈和开发边界

完成标准：

- 文档结构完整
- 一期范围明确
- 关键角色和关键业务无缺失

验证方式：

- 文档评审
- 需求范围核对
- 角色和流程覆盖检查

### 第二阶段：工程基线阶段

工作内容：

- 创建后端 Spring Boot 工程
- 创建前端 Vue 3 工程
- 创建数据库脚本目录和初始化脚本
- 建立 README、开发规范和协作约束

完成标准：

- 前后端目录结构清晰
- 后端具备可启动入口
- 前端具备可访问页面入口
- 数据库具备初始化脚本

验证方式：

- 后端启动验证
- 前端启动验证
- 数据库脚本执行验证

### 第三阶段：系统底座阶段

工作内容：

- 登录与认证
- 用户、组织、角色、岗位管理
- 菜单权限和数据权限
- 字典和系统参数
- 附件中心
- 消息中心
- 操作日志

完成标准：

- 管理员可维护基础数据
- 菜单和接口权限可生效
- 附件和消息能力可复用

验证方式：

- 管理员功能验证
- 权限校验验证
- 附件上传和消息通知验证

### 第四阶段：流程中心阶段

工作内容：

- 流程定义
- 节点定义
- 流程实例
- 待办、已办、抄送
- 审批、退回、撤回、转交
- 流程轨迹展示

完成标准：

- 通用流程引擎可支撑多个业务场景
- 流程状态和审批轨迹完整

验证方式：

- 用两个以上业务跑通同一套流程能力
- 检查退回、重提、办结等状态流转

### 第五阶段：一期重点业务阶段

工作内容：

- 学生请假与销假
- 实习材料管理
- 异常学生材料管理
- 教师调课
- 课程标准评审
- 课题申报
- 会议室申请
- 维修工单

完成标准：

- 业务表单可填写
- 审批流程可流转
- 附件可上传
- 结果可查询
- 基础统计可查看

验证方式：

- 按角色逐一验证流程闭环
- 验证业务状态和流程状态一致性

### 第六阶段：报表与优化阶段

工作内容：

- 工作台待办统计
- 提交率和审批时效报表
- 异常与超时预警
- 页面体验优化
- 代码结构整理

完成标准：

- 报表与台账能对上
- 超时和异常可识别
- 常用页面操作顺畅

验证方式：

- 报表核对
- 超时任务验证
- 页面联调测试

## 7. 每一步如何验证完成

### 文档阶段

- 检查文档是否覆盖背景、目标、角色、流程、数据、权限和验收标准

### 工程阶段

- 检查前后端是否可启动
- 检查数据库脚本是否可执行

### 基础平台阶段

- 用管理员账户验证组织、用户、角色和菜单维护
- 用不同角色验证权限隔离

### 流程阶段

- 验证提交、审批、退回、撤回、转交、办结
- 验证待办和已办统计

### 业务阶段

- 按真实业务场景走通表单、附件、审批、查询、统计

### 报表阶段

- 核对统计结果与业务台账
- 核对异常和超时提示是否准确

## 9. 风险控制

- 需求蔓延：用优先级和一期边界控制
- 流程差异：通过可配置流程模板处理
- 权限复杂：通过角色矩阵和数据权限规则提前设计
- 附件过多：统一附件中心
- 多部门协同：通过待办、抄送、催办和消息中心增强可见性

## 10. 当前阶段进展补充

截至当前批次，项目在“系统底座阶段”和“流程中心阶段”已补齐以下一期核心能力：

- 用户管理：支持用户列表、新增、编辑、启停用、角色分配
- 组织管理：支持组织树查询、新增、编辑
- 角色权限管理：支持角色维护、菜单分配、按钮权限分配
- 附件中心：支持按申请单上传、列表查看、下载、删除
- 流程模板管理：支持流程模板查询、新增、编辑和节点配置

当前实现仍遵循一期最小可用原则：

- 使用现有 Spring Boot + Vue 3 + MySQL 结构继续开发
- 权限和流程能力优先满足“可配置、可演示、可验证”
- 不在本阶段引入复杂流程设计器、重量级工作流引擎或额外中间件
