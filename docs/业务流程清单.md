# 校园 OA 业务流程清单

以下清单面向一期与后续扩展，覆盖学生事务、教学教务、科研管理、行政后勤等业务域。每个业务均给出参与角色、当前痛点、信息化目标、流程步骤、表单字段、审批规则和优先级。

---

## 1. 学生请假申请

- 业务名称：学生请假申请
- 参与角色：学生、班主任/辅导员、学院审批人
- 当前痛点：线下请假单流转慢，学生无法查看进度，请假台账统计困难
- 信息化目标：实现线上申请、线上审批、请假留痕和统计汇总
- 流程步骤：学生填写请假信息并上传附件 -> 班主任审批 -> 超过阈值进入学院审批 -> 审批通过后生成请假记录
- 表单字段：姓名、学号、班级、请假类型、开始时间、结束时间、请假天数、请假原因、联系电话、附件
- 审批规则：短期请假由班主任审批，长期请假或特殊类型请假需学院审批
- 优先级：P1

## 2. 学生销假申请

- 业务名称：学生销假申请
- 参与角色：学生、班主任/辅导员
- 当前痛点：返校后销假依赖口头确认，台账状态不准确
- 信息化目标：将请假和返校闭环打通
- 流程步骤：学生提交销假申请 -> 班主任确认返校情况 -> 更新请假记录状态
- 表单字段：关联请假单号、返校时间、实际返校说明、附件
- 审批规则：由原审批链中的班主任或辅导员确认销假
- 优先级：P1

## 3. 实习材料提交

- 业务名称：实习材料提交
- 参与角色：学生、班主任、学院、教务处
- 当前痛点：材料分散、催交困难、版本混乱
- 信息化目标：统一收集和归档实习材料
- 流程步骤：学生上传实习协议和相关材料 -> 班主任审核完整性 -> 学院汇总 -> 教务处备案
- 表单字段：学生姓名、学号、班级、实习单位、岗位、开始日期、结束日期、材料类型、附件
- 审批规则：班主任先审完整性，学院负责汇总确认，教务处只读备案或抽查
- 优先级：P1

## 4. 异常学生材料管理

- 业务名称：异常学生材料管理
- 参与角色：班主任、辅导员、学院
- 当前痛点：异常学生跟踪记录分散，干预过程缺少留痕
- 信息化目标：建立异常学生电子档案和跟踪闭环
- 流程步骤：班主任建档 -> 填写异常情况和谈话记录 -> 学院复核 -> 后续跟踪 -> 闭环归档
- 表单字段：学生信息、异常类型、异常级别、异常说明、谈话记录、处理措施、附件
- 审批规则：班主任发起，学院复核，重大异常可增加领导审批
- 优先级：P1

## 5. 奖学金申请

- 业务名称：奖学金申请
- 参与角色：学生、班主任、学院、学工处
- 当前痛点：材料收集依赖班级人工统计，评审透明度低
- 信息化目标：线上申请、审核、评审与公示留痕
- 流程步骤：学生提交申请 -> 班主任初审 -> 学院评审 -> 学工处备案/审批
- 表单字段：学生信息、奖项类型、成绩排名、获奖情况、家庭情况、附件
- 审批规则：班主任初审，学院评审，部分奖项由学工处终审
- 优先级：P2

## 6. 助学金申请

- 业务名称：助学金申请
- 参与角色：学生、班主任、学院、学工处
- 当前痛点：贫困材料和申请材料重复提交，审批慢
- 信息化目标：统一材料、统一审核、统一统计
- 流程步骤：学生提交助学金申请 -> 班主任核实 -> 学院审核 -> 学工处审批
- 表单字段：学生信息、家庭经济情况、困难等级、申请理由、附件
- 审批规则：需与困难认定结果关联，班主任和学院逐级审核
- 优先级：P2

## 7. 困难认定申请

- 业务名称：困难认定申请
- 参与角色：学生、班主任、学院、学工处
- 当前痛点：困难等级认定材料多、审批周期长
- 信息化目标：形成统一困难认定档案
- 流程步骤：学生提交申请 -> 班主任核实 -> 学院审核 -> 学工处确认
- 表单字段：学生信息、家庭成员情况、收入情况、特殊情况说明、附件
- 审批规则：按照学校困难认定标准分级审核
- 优先级：P2

## 8. 在读证明申请

- 业务名称：在读证明申请
- 参与角色：学生、班主任、学院
- 当前痛点：重复线下盖章、开具记录不完整
- 信息化目标：实现申请、审核和开具留痕
- 流程步骤：学生申请 -> 班主任核实 -> 学院开具 -> 学生下载或打印
- 表单字段：学生信息、证明用途、接收单位、备注
- 审批规则：普通证明由班主任/学院办理，特殊证明可加学院审批
- 优先级：P2

## 9. 学生离校申请

- 业务名称：学生离校申请
- 参与角色：学生、班主任、学院、宿管
- 当前痛点：节假日离校和返校统计不及时
- 信息化目标：统一离校备案和返校确认
- 流程步骤：学生申请离校 -> 班主任审批 -> 宿舍信息同步 -> 学生返校后确认
- 表单字段：学生信息、离校原因、离校时间、返校时间、去向、联系人
- 审批规则：班主任审批为主，特殊时段需学院审批
- 优先级：P2

## 10. 学生返校确认

- 业务名称：学生返校确认
- 参与角色：学生、班主任、宿管
- 当前痛点：返校状态难追踪，存在统计滞后
- 信息化目标：实现返校闭环和宿舍联动
- 流程步骤：学生提交返校确认 -> 宿管登记 -> 班主任确认
- 表单字段：学生信息、返校时间、宿舍信息、备注
- 审批规则：返校确认由班主任或宿管协同完成
- 优先级：P3

## 11. 教师调课申请

- 业务名称：教师调课申请
- 参与角色：任课教师、教研室、学院、教务处
- 当前痛点：调课通知容易遗漏，教室和时间冲突难管控
- 信息化目标：统一调课审批和通知
- 流程步骤：教师申请调课 -> 教研室审核 -> 学院审批 -> 教务处确认并通知
- 表单字段：教师姓名、课程名称、原上课时间、调整后时间、教室、原因、附件
- 审批规则：涉及跨教学周或重要课程需教务处终审
- 优先级：P1

## 12. 停课申请

- 业务名称：停课申请
- 参与角色：任课教师、教研室、学院、教务处
- 当前痛点：停课审批不统一，补课安排缺乏联动
- 信息化目标：将停课与后续补课联动管理
- 流程步骤：教师提交停课申请 -> 教研室审核 -> 学院审批 -> 教务处备案
- 表单字段：课程信息、停课时间、停课原因、补课计划、附件
- 审批规则：需同时填写补课安排建议
- 优先级：P2

## 13. 补课申请

- 业务名称：补课申请
- 参与角色：任课教师、教研室、学院、教务处
- 当前痛点：补课安排通知不及时，教室资源冲突
- 信息化目标：统一补课时间、教室和通知管理
- 流程步骤：教师提交补课申请 -> 教研室审核 -> 学院审批 -> 教务处确认
- 表单字段：课程信息、补课时间、补课地点、补课原因、关联停课单
- 审批规则：必须校验教室和时间冲突
- 优先级：P2

## 14. 课程标准评审

- 业务名称：课程标准评审
- 参与角色：任课教师、教务处审核角色、学院
- 当前痛点：文档收集和评审意见留痕不足
- 信息化目标：实现课程标准提交、评审、归档一体化
- 流程步骤：教师提交课程标准 -> 一期由教务处审核角色评审 -> 形成意见 -> 学院备案
- 表单字段：课程编号、课程名称、适用专业、学时学分、版本号、附件、评审意见
- 审批规则：一期先由教务处审核角色跑通流程，后续再扩展教研室多评审人机制
- 优先级：P1

## 15. 教材征订申请

- 业务名称：教材征订申请
- 参与角色：任课教师、教研室、学院、教务处
- 当前痛点：教材清单分散，汇总靠 Excel 手工拼接
- 信息化目标：统一征订填报和汇总
- 流程步骤：教师填报教材 -> 教研室审核 -> 学院汇总 -> 教务处确认
- 表单字段：课程名称、教材名称、ISBN、出版社、作者、使用班级、数量、备注
- 审批规则：重复教材自动提示，学院汇总后提交教务处
- 优先级：P2

## 16. 教案提交

- 业务名称：教案提交
- 参与角色：任课教师、教研室
- 当前痛点：教案版本散乱，检查困难
- 信息化目标：统一提交和归档教学资料
- 流程步骤：教师上传教案 -> 教研室检查 -> 退回修改或通过归档
- 表单字段：课程名称、学期、章节范围、版本、附件、说明
- 审批规则：教研室审核通过后归档
- 优先级：P3

## 17. 教学大纲提交

- 业务名称：教学大纲提交
- 参与角色：任课教师、教研室、学院
- 当前痛点：教学大纲更新频率高，版本难控
- 信息化目标：实现大纲版本管理和统一备案
- 流程步骤：教师提交 -> 教研室审核 -> 学院备案
- 表单字段：课程信息、适用专业、版本号、修订说明、附件
- 审批规则：教研室审核后进入学院备案
- 优先级：P3

## 18. 成绩更正申请

- 业务名称：成绩更正申请
- 参与角色：任课教师、学院、教务处
- 当前痛点：成绩更正审批链长，材料难追溯
- 信息化目标：实现更正申请全过程留痕
- 流程步骤：教师发起更正 -> 学院审核 -> 教务处审批 -> 更新状态
- 表单字段：课程信息、学生信息、原成绩、新成绩、更正原因、证明材料
- 审批规则：必须上传证明材料，教务处终审
- 优先级：P2

## 19. 考试安排申请

- 业务名称：考试安排申请
- 参与角色：学院、教务处
- 当前痛点：考试安排和教室、监考协调复杂
- 信息化目标：形成标准化考试安排台账
- 流程步骤：学院提交考试需求 -> 教务处统筹排考 -> 生成安排通知
- 表单字段：课程名称、班级、考试人数、考试时间建议、教室需求、备注
- 审批规则：由教务处统一审核并分配资源
- 优先级：P3

## 20. 教室借用申请

- 业务名称：教室借用申请
- 参与角色：教师、学院、教务处/管理员
- 当前痛点：教室使用冲突多，审批记录不统一
- 信息化目标：统一借用申请和冲突控制
- 流程步骤：申请人提交 -> 所属学院审批 -> 管理员确认教室资源
- 表单字段：借用时间、教室类型、人数、用途、设备需求、附件
- 审批规则：必须校验时间和场地冲突
- 优先级：P2

## 21. 毕业设计开题申请

- 业务名称：毕业设计开题申请
- 归属说明：归入学生生命周期业务，由学生发起，指导老师/教研相关角色审批。
- 参与角色：学生、指导教师、教研室、学院
- 当前痛点：开题材料分散，过程节点不统一
- 信息化目标：将毕业设计过程纳入统一管理
- 流程步骤：学生提交开题材料 -> 指导教师审核 -> 教研室审核 -> 学院备案
- 表单字段：题目、学生信息、指导教师、研究内容、进度计划、附件
- 审批规则：指导教师同意后方可进入教研室审核
- 优先级：P3

## 22. 毕业设计中期检查

- 业务名称：毕业设计中期检查
- 归属说明：归入学生生命周期业务，由学生发起，指导老师/教研相关角色审批。
- 参与角色：学生、指导教师、教研室
- 当前痛点：进度检查不统一，整改记录不完整
- 信息化目标：沉淀中期检查记录和整改结果
- 流程步骤：学生提交中期材料 -> 指导教师审核 -> 教研室检查 -> 反馈整改
- 表单字段：论文进度、阶段成果、存在问题、整改计划、附件
- 审批规则：中期未通过需重新提交整改结果
- 优先级：P3

## 23. 课题申报

- 业务名称：课题申报
- 参与角色：教师、学院、科技处、评审专家
- 当前痛点：申报材料多、评审过程分散
- 信息化目标：建立统一申报和评审平台
- 流程步骤：教师提交申报书 -> 学院审核 -> 科技处分配评审 -> 汇总结果 -> 立项归档
- 表单字段：项目名称、项目类别、负责人、成员、经费预算、研究内容、附件
- 审批规则：学院初审，科技处组织评审，专家评分后汇总
- 优先级：P1

## 24. 课题中期检查

- 业务名称：课题中期检查
- 参与角色：教师、学院、科技处
- 当前痛点：进展材料催交困难，检查口径不一
- 信息化目标：跟踪项目执行进度
- 流程步骤：负责人提交中期报告 -> 学院审核 -> 科技处检查 -> 反馈结果
- 表单字段：项目名称、执行进度、阶段成果、经费使用说明、附件
- 审批规则：科技处拥有最终检查结论
- 优先级：P2

## 25. 课题结题申请

- 业务名称：课题结题申请
- 参与角色：教师、学院、科技处、评审专家
- 当前痛点：结题材料繁多，专家意见归档不完整
- 信息化目标：结题材料、评审意见和结题结果统一归档
- 流程步骤：教师提交结题材料 -> 学院审核 -> 科技处组织专家评审 -> 形成结论
- 表单字段：项目名称、成果列表、经费说明、结题报告、附件
- 审批规则：需专家评审通过后方可结题
- 优先级：P2

## 26. 科研成果登记

- 业务名称：科研成果登记
- 参与角色：教师、学院、科技处
- 当前痛点：论文、专利、著作分散登记，统计困难
- 信息化目标：形成统一成果台账
- 流程步骤：教师填报成果 -> 学院审核 -> 科技处确认入库
- 表单字段：成果类型、名称、作者、发表或授权时间、级别、附件
- 审批规则：学院审核真实性，科技处确认入库
- 优先级：P3

## 27. 学术讲座申请

- 业务名称：学术讲座申请
- 参与角色：教师、学院、科技处
- 当前痛点：讲座审批和备案信息零散
- 信息化目标：实现讲座申请、审批和备案统一管理
- 流程步骤：申请人提交 -> 学院审核 -> 科技处备案/审批
- 表单字段：讲座主题、主讲人、时间地点、参与范围、预算、附件
- 审批规则：跨学院或校级活动需科技处审批
- 优先级：P3

## 28. 会议室申请

- 业务名称：会议室申请
- 参与角色：教师、部门人员、管理员
- 当前痛点：预约靠群消息，容易重复占用
- 信息化目标：实现会议室预约、审批、排期和冲突控制
- 流程步骤：申请人提交预约 -> 管理员审批 -> 系统锁定时段 -> 使用结束后归档
- 表单字段：申请人、部门、会议主题、时间、参会人数、设备需求、备注
- 审批规则：按会议室类型或归属部门分配审批人
- 优先级：P1

## 29. 物资借用申请

- 业务名称：物资借用申请
- 参与角色：学生、教师、管理员
- 当前痛点：借用登记纸质化，归还状态不清晰
- 信息化目标：统一借用、审批、归还和台账管理
- 流程步骤：提交借用申请 -> 管理员审批 -> 发放物资 -> 归还确认
- 表单字段：借用物品、数量、用途、借用时间、归还时间、联系人
- 审批规则：贵重物品可增加部门负责人审批
- 优先级：P2

## 30. 办公用品申领

- 业务名称：办公用品申领
