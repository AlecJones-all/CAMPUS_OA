# 第六批业务开发说明

## 范围

本批次按 `$campus-oa-batch-5` 流程开发 5 个业务：

- 学术讲座申请：`academic-lecture-applications`
- 办公用品申领：`office-supply-requests`
- 实验室安全隐患上报：`lab-safety-hazard-reports`
- 宿舍调宿申请：`dorm-adjustment-requests`
- 用章申请：`stamp-requests`

## 角色与流程

| 业务 | 发起角色 | 审批角色 | 流程类型 |
| --- | --- | --- | --- |
| 学术讲座申请 | 教师 | 科技处 | `ACADEMIC_LECTURE_APPLICATION` |
| 办公用品申领 | 教师 | 管理员 | `OFFICE_SUPPLY_REQUEST` |
| 实验室安全隐患上报 | 教师 | 管理员 | `LAB_SAFETY_HAZARD_REPORT` |
| 宿舍调宿申请 | 学生 | 班主任 | `DORM_ADJUSTMENT_REQUEST` |
| 用章申请 | 教师 | 管理员 | `STAMP_REQUEST` |

当前系统工作流引擎采用单审批节点模型，提交后进入对应角色审批；退回、撤回、审批记录和归档复用统一工作流能力。

## 数据表

- `biz_academic_lecture_application`
- `biz_office_supply_request`
- `biz_lab_safety_hazard_report`
- `biz_dorm_adjustment_request`
- `biz_stamp_request`

历史数据库需执行：

```sql
source database/mysql/003_sync_batch6_businesses.sql;
```

新初始化数据库已同步到 `database/mysql/001_init_schema.sql`。
