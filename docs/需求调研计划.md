# 校园 OA 需求调研计划

## 1. 调研目标

- 全面梳理校园 OA 涉及的业务场景、参与角色和审批链
- 识别当前线下流程中的主要痛点和信息化机会
- 明确一期建设范围、优先级和边界
- 为后续原型、数据库设计、接口设计和开发计划提供依据

## 2. 调研原则

- 先高频后低频
- 先共性后个性
- 先真实业务再技术实现
- 先流程闭环再局部优化

## 3. 调研对象

- 学生
- 班主任
- 辅导员
- 任课教师
- 教研室主任/秘书
- 学院教学秘书
- 学院领导
- 教务处
- 科技处
- 学工处
- 后勤处
- 宿管中心
- 资产管理部门
- 实验室管理部门
- 系统管理员

## 4. 调研内容

### 4.1 组织与角色

- 学校组织层级如何划分
- 角色如何对应实际岗位
- 是否存在兼岗
- 不同角色的数据边界如何划分

### 4.2 业务流程

- 当前流程发起方式
- 审批顺序和审批条件
- 是否存在退回、补交、撤回、转交
- 是否有超时要求
- 是否需要归档或导出

### 4.3 表单与材料

- 每项业务需要填写哪些字段
- 需要上传哪些附件
- 哪些字段必须校验
- 是否存在模板、编号、打印需求

### 4.4 统计与预警

- 各部门关心哪些统计口径
- 是否有日报、周报、月报需求
- 哪些业务需要超时提醒或异常预警

## 5. 调研方式

### 5.1 访谈

- 与各业务部门进行面对面或线上访谈
- 重点了解当前做法、痛点、特殊规则和历史遗留问题

### 5.2 文档收集

- 收集现有纸质表单
- 收集 Excel 台账
- 收集审批单、通知单、模板文件
- 收集业务管理制度和管理办法

### 5.3 流程梳理

- 绘制泳道图
- 标记发起、审批、退回、归档节点
- 标记异常处理路径

### 5.4 数据抽样

- 抽取近一学期或一学年真实业务样本
- 统计业务量、审批时长、异常率、补交率

## 6. 调研输出物

- 需求分析文档
- 业务流程清单
- 角色权限说明
- 数据库设计草案
- 开发计划
- 后续补充的原型和接口设计文档

## 7. 调研阶段安排

### 第一阶段：准备阶段

- 收集现有制度文件和表单模板
- 确定调研对象和访谈顺序
- 建立统一调研记录模板

验收方式：
- 形成调研名单、问题清单和资料清单

### 第二阶段：业务访谈阶段

- 分角色访谈学生、教师、学院、职能部门
- 记录业务过程中的真实痛点和特例

验收方式：
- 每个部门至少形成一份调研记录

### 第三阶段：流程归纳阶段

- 对同类业务进行归并
- 抽象共性流程和公共能力
- 标注一期优先级

验收方式：
- 输出统一业务清单和流程分类结果

### 第四阶段：需求定稿阶段

- 形成正式需求文档
- 形成角色权限矩阵
- 形成开发路线和验证方式

验收方式：
- 文档可进入设计和开发阶段

## 8. 调研重点问题清单

- 哪些业务是高频且跨部门的
- 哪些业务附件最多、最难收集
- 哪些业务审批最容易卡住
- 哪些业务最需要统计报表
- 哪些业务最容易发生越权查看
- 哪些业务需要长期归档
- 哪些业务存在学院之间差异化规则

## 9. 一期建议调研优先级

### 第一优先级

- 请假销假
- 实习材料
- 异常学生
- 教师调课
- 课程标准评审
- 课题申报
- 会议室申请
- 维修工单

### 第二优先级

- 奖助学金
- 教材征订
- 成绩更正
- 物资借用
- 证明开具
- 实验室安全

### 第三优先级

- 车辆申请
- 用章申请
- 学术活动备案
- 横向项目归档
