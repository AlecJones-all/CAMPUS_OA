# 批次 5 运行库升级说明

## 背景

2026-04-26 对以下 5 个业务进行联调时，在线后端接口已能返回更明确的数据库错误：

- 学生离校申请
- 学生返校确认
- 毕业设计开题申请
- 毕业设计中期检查
- 科研成果登记

实际排查结果表明，当前 `campus_oa` 运行库仍停留在旧批次结构，缺少这 5 个业务对应的新表和流程类型种子，因此会导致：

- 列表接口返回 500
- 选项接口返回 500
- 创建流程即使请求体正确，也会因为找不到流程类型或业务表而失败

## 已确认缺失项

运行库缺少以下业务表：

- `biz_student_leave_application`
- `biz_student_return_confirmation`
- `biz_graduation_project_opening`
- `biz_graduation_project_midterm`
- `biz_research_achievement_registration`

运行库缺少以下流程类型：

- `STUDENT_LEAVE_APPLICATION`
- `STUDENT_RETURN_CONFIRMATION`
- `GRADUATION_PROJECT_OPENING`
- `GRADUATION_PROJECT_MIDTERM`
- `RESEARCH_ACHIEVEMENT_REGISTRATION`

运行库缺少以下流程定义：

- `WF_STUDENT_LEAVE_APPLICATION`
- `WF_STUDENT_RETURN_CONFIRMATION`
- `WF_GRADUATION_PROJECT_OPENING`
- `WF_GRADUATION_PROJECT_MIDTERM`
- `WF_RESEARCH_ACHIEVEMENT_REGISTRATION`

## 升级方式

在历史库环境中，不应重新执行全量初始化脚本覆盖已有数据，应执行增量脚本：

- [002_sync_batch5_businesses.sql](/D:/campus_oa/database/mysql/002_sync_batch5_businesses.sql)

执行命令示例：

```powershell
cmd /c mysql -uroot -p123456789 -D campus_oa < D:\campus_oa\database\mysql\002_sync_batch5_businesses.sql
```

## 升级后预期

升级完成后，以下接口应从 500 恢复为正常响应：

- `GET /api/business/student-leave-applications`
- `GET /api/business/student-return-confirmations`
- `GET /api/business/student-return-confirmations/options/relatedLeaveNo`
- `GET /api/business/graduation-project-midterms`
- `GET /api/business/graduation-project-midterms/options/projectName`
- `GET /api/business/research-achievement-registrations`

如果业务数据为空，接口可以返回空数组，但不应再返回数据库结构错误。

## 配套代码

后端已补充运行库自检：

- [DatabaseSchemaInspector.java](/D:/campus_oa/backend/src/main/java/com/campusoa/common/DatabaseSchemaInspector.java)

当前自检不仅检查表和列，也会检查这 5 个业务的流程类型与流程定义是否存在。
