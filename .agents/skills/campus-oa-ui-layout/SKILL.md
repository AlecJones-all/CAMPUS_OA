## UI 优化目标

1. 页面布局统一。
2. 菜单层级清晰。
3. 功能入口不堆叠。
4. 按钮数量合理。
5. 重要操作突出，次要操作收纳。
6. 内容少的页面不能显得空。
7. 内容多的页面不能显得臃肿。
8. 适合课程项目演示和答辩展示。

## 页面布局规范

### 1. 列表页

列表页建议包含：

- 页面标题
- 简短说明
- 查询筛选区
- 主要操作按钮
- 数据表格
- 状态标签
- 分页区域

规则：

1. 主操作按钮最多 1-2 个，例如“新建申请”“导出”。
2. 行内按钮不要太多，最多直接展示 2 个。
3. 多余操作放入“更多”下拉菜单。
4. 查询条件超过 4 个时，使用折叠高级筛选。
5. 状态使用 Tag 展示，例如草稿、待审批、已通过、已驳回。

### 2. 表单页

表单页建议包含：

- 页面标题
- 业务说明
- 表单分组
- 附件区域
- 提交按钮区

规则：

1. 表单字段多时按模块分组。
2. 使用 Card 分区。
3. 底部固定操作区或统一右下角按钮。
4. 主按钮只有一个，例如“提交申请”。
5. 次按钮包括“保存草稿”“返回”。

### 3. 详情页

详情页建议包含：

- 基本信息
- 表单内容
- 附件材料
- 审批记录时间线
- 当前状态
- 操作区

规则：

1. 详情页不要堆太多按钮。
2. 审批操作只在有权限时显示。
3. 审批通过、驳回放在统一审批区域。
4. 审批记录使用 Timeline 或表格。

### 4. 仪表盘

仪表盘建议包含：

- 待办数量
- 我的申请数量
- 已通过数量
- 已驳回数量
- 常用业务入口
- 最近申请记录

规则：

1. 不要把全部功能都放首页。
2. 首页只放常用入口。
3. 功能较少时用卡片补充说明。
4. 功能较多时使用分组。

## 按钮数量控制规则

1. 页面顶部只放核心操作。
2. 表格行内只放：
   - 查看
   - 处理 / 审批
3. 其他操作放入更多菜单：
   - 编辑
   - 撤回
   - 删除
   - 导出
   - 查看日志
4. 危险操作必须二次确认。
5. 删除类按钮不直接暴露给普通用户。

## 菜单设计规则

菜单建议按业务分组：

1. 工作台
2. 我的申请
3. 我的待办
4. 学生事务
5. 教学事务
6. 科研事务
7. 后勤事务
8. 系统管理

角色显示建议：

- 学生：工作台、我的申请、新建申请
- 班主任：工作台、我的待办、学生事务
- 教研室：工作台、我的待办、教学事务
- 科技处：工作台、我的待办、科研事务
- 管理员：全部菜单

## 内容少的页面处理

如果页面内容少，不要空白太多，可以增加：

1. 功能说明卡片
2. 操作引导
3. 最近记录
4. 常见问题
5. 状态统计卡片
6. 空状态插图或 Empty 组件

## 内容多的页面处理

如果页面内容多，必须使用：

1. 分组 Card
2. Tabs
3. 折叠面板
4. 高级筛选折叠
5. 更多操作下拉
6. 分页
7. 详情弹窗或详情页
