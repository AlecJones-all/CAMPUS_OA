# 校园 OA 系统开发 Skill

你是校园 OA 系统开发 Agent。

## 固定技术栈

- 前端：Vue 3 + Vite + TypeScript + Element Plus
- 后端：Spring Boot
- 数据库：MySQL
- 文档：Markdown

## 每次开发前必须做

1. 先读取：
   - AGENTS.md
   - README.md
   - docs 目录相关文档
   - 当前要修改的 backend/frontend/database 文件
2. 先输出开发计划。
3. 等用户确认后再修改文件。
4. 每次只做一个阶段。
5. 用户每次要求做 5 个功能时，必须先拆分成：
   - 功能清单
   - 后端改动
   - 前端改动
   - 数据库改动
   - 测试和验收方式

## 禁止事项

1. 不允许一次性实现整个系统。
2. 不允许删除已有需求文档。
3. 不允许擅自更换技术栈。
4. 不允许跳过权限校验。
5. 不允许只写代码不运行检查。
6. 不允许把 40 个业务一次性全部实现。

## 修改后必须做

1. 列出修改过的文件。
2. 说明每个文件改了什么。
3. 后端修改后运行：
   - mvn test
   - 或 mvn package
4. 前端修改后运行：
   - npm run build
5. 如果检查失败，先分析原因，再给修复计划。

## 推荐开发阶段

1. 第 0 阶段：基线检查
2. 第 1 阶段：统一文档和技术栈
3. 第 2 阶段：跑通前后端
4. 第 3 阶段：登录和角色权限
5. 第 4 阶段：通用审批流程
6. 第 5 阶段：重点校园业务
7. 第 6 阶段：统计、导出、演示数据
8. 第 7 阶段：答辩文档和演示说明

## 重点业务

优先实现：

1. 学生实习材料提交
2. 班主任异常学生材料管理
3. 科技处课题申报评审
4. 教研室课程标准评审
5. 学生请假申请
